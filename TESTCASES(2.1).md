# API Тест-кейсы
# POST /api/1/item:

### TC-001: Успешное создание объявления
**Шаги:** 
1. Отправить POST-запрос на `/api/1/item` .
2. В теле запроса передать:
```json
{
  "sellerID": 143456,
  "name": "Игровая приставка PlayStation 5",
  "price": 45000,
  "statistics": {
    "likes": 2,
    "viewCount": 1,
    "contacts": 10
  }
}
```
**Ожидаемый результат:** 200 OK. 

### TC-002: Создание объявления с пустым именем
**Шаги:** 
1. Отправить POST-запрос на `/api/1/item` с пустым name.
2. В теле запроса передать:
```json
{
  "sellerID": 143736,
  "name": "",
  "price": 6000,
  "statistics": {
    "likes": 1,
    "viewCount": 1,
    "contacts": 2
  }
}
```
**Ожидаемый результат:** 400 Bad Request

### TC-003: Создание объявления с отрицательным прайсом
**Шаги:** 
1. Отправить POST-запрос на `/api/1/item` с price=-100.
2. В теле запроса передать:
```json
{
  "sellerID": 134736,
  "name": "Кукла",
  "price": -100,
  "statistics": {
    "likes": 1,
    "viewCount": 1,
    "contacts": 2
  }
}
```
**Ожидаемый результат:** 400 Bad Request.

### TC-004: Идемпотентность
**Шаги:** 
1. Дважды отправить POST-запрос на `/api/1/item` с одним телом.
2. В теле запроса передать: 
```json
{
  "sellerID": 999999,
  "name": "Машинка",
  "price": 200,
  "statistics": {
    "likes": 1,
    "viewCount": 1,
    "contacts": 2
  }
}
```
**Ожидаемый результат:** 200 OK. Каждый раз создается уникальный id

### TC-005: Создание объявления с неверным типом данным
**Шаги:** 
1. Отправить POST-запрос на `/api/1/item` с sellerID="один"
2. В теле запроса передать: 
```json
{
  "sellerID": "один",
  "name": "Машинка",
  "price": 200,
  "statistics": {
    "likes": 1,
    "viewCount": 1,
    "contacts": 2
  }
}
```
**Ожидаемый результат:** 400 Bad Request.

# GET /api/1/item/{id}:

### TC-101: Получение существующего объявления
**Шаги:** 
1. Отправить GET-запрос на `/api/1/item/{id}` (где {id} — существующий UUID)

**Ожидаемый результат:** 200 OK. Тело ответа содержит:  createdAt, id, sellerId, name, price, statistics.

### TC-102: Получение несуществующего объявления
**Шаги:** 
1. Отправить GET-запрос на  `/api/1/item/e4a685e1-b27b-4954-9540-66726fc6e73d`

**Ожидаемый результат:** 404 Not Found.

### TC-103: Получение объявления с невалидным ID
**Шаги:** 
1. Отправить GET-запрос на `/api/1/item/abc` (строка вместо UUID)

**Ожидаемый результат:** 400 Bad Request.

### TC-104: Получение удалённого объявления
**Шаги:** 
1. Создать объявление через POST, сохранить UUID
2. Удалить объявление через DELETE `/api/2/item/{id}`
3. Отправить GET-запрос на `/api/1/item/{id}`

**Ожидаемый результат:** 404 Not Found.

# GET /api/1/{sellerID}/item:

### TC-201: Успешное получение всех объявлений продавца
**Шаги:** 
1. Отправить GET-запрос на `/api/1/{sellerID}/item` (где у продавца есть объявления)

**Ожидаемый результат:** 200 OK.
Тело ответа — массив объявлений.
Все объявления принадлежат `{sellerID}`.

### TC-202: Несуществующий продавец с sellerID=9999989
**Шаги:** 
1. Отправить GET-запрос на `/api/1/9999989/item`

**Ожидаемый результат:** 200 OK.
Пустой массив.

### TC-203: Неверный формат sellerID
**Шаги:** 
1. Отправить GET-запрос на `/api/1/abc/item` (строка вместо числа)

**Ожидаемый результат:** 400 Bad Request.

# GET /api/1/statistic/{id} и /api/2/statistic/{id}:

### TC-301: Успешное получение статистики (v1)
**Шаги:** 
1. Отправить GET-запрос на `/api/1/statistic/{id}` (где {id} — существующий UUID)

**Ожидаемый результат:** 200 OK.
Тело ответа содержит: likes, viewCount, contacts.

### TC-302: Успешное получение статистики (v2)
**Шаги:** 
1. Отправить GET-запрос на `/api/2/statistic/{id}` (где {id} — существующий UUID)

**Ожидаемый результат:** 200 OK.
Тело ответа содержит: likes, viewCount, contacts.

### TC-303: Получение статистики несуществующего объявления
**Шаги:** 
1. Отправить GET-запрос на `/api/2/statistic/{id}`и `/api/1/statistic/{id}` (где {id} — несуществующий UUID)

**Ожидаемый результат:** 404 Not Found.

### TC-304: Получение статистики удалённого объявления
**Шаги:** 
1. Создать объявление, сохранить UUID
2. Удалить через DELETE `/api/2/item/{id}` (где {id} —  UUID)
3. Отправить GET-запрос на `/api/1/statistic/{id}` (где {id} —  UUID)

**Ожидаемый результат:** 404 Not Found.
