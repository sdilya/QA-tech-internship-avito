# BUG-001: API принимает отрицательную цену
**Краткое описание:**

Эндпоинт `POST /api/1/item` не валидирует поле `price`. Сервер успешно создаёт объявление с отрицательной стоимостью, что нарушает бизнес-логику сервиса.

**Шаги воспроизведения:**
1. Отправить POST-запрос на `/api/1/item`
2. В теле запроса передать:
```json
{
  "sellerID": 134736,
  "name": "Кукла",
  "price": -100,
  "statistics": {
    "likes": 1,
    "viewCount": 1,
    "contacts": 2
  }
}
```

**Фактический результат:**

Статус код: 200 OK. 

Поле price сохранено как -100

**Ожидаемый результат:**

Статус код: 400 Bad Request.

Тело ответа содержит ошибку валидации (например, "Цена не должна быть отрицательной").

Объявление не создаётся.

**Серьёзность:**

Major 

**Приоритет:**

P1 

**Окружение:**
1. ОС: Windows 11
2. Python: 3.13.7
3. Библиотеки: pytest 7.4.0, requests 2.32.0
4. Дата: 2026-03-25

# BUG-002: Статистика доступна для удалённого объявления
**Краткое описание:**

После удаления объявления через `DELETE /api/2/item/{id}`, эндпоинт получения статистики `GET /api/1/statistic/{id}` продолжает возвращать данные (200 OK) вместо ожидаемой ошибки 404.

**Шаги воспроизведения:**
1. Создать объявление через `POST /api/1/item`, сохранить UUID
2. Удалить это объявление: `DELETE /api/2/item/{id}`
3. Убедиться, что данное объявление пользователя удалено: `GET /api/1/{sellerID}/item` 
4. Запросить статистику удалённого объявления: `GET /api/1/statistic/{id}`

**Фактический результат:**

Статус код: 200 OK.

Тело ответа содержит массив со статистикой:
```json
[
  {
    "likes": 1,
    "viewCount": 1,
    "contacts": 1
  }
]
```

**Ожидаемый результат:**

Статус код: 404 Not Found.

Тело ответа содержит сообщение об ошибке (например, "Объявление не найдено")

**Серьёзность:**

Minor

**Приоритет:** 

P2 

**Окружение:**
1. ОС: Windows 11
2. Python: 3.13.7
3. Библиотеки: pytest 7.4.0, requests 2.32.0
