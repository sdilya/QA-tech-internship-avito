# BUG-001: Фильтр диапазона цен не фильтрует объявления
**Краткое описание:**

При применении фильтра по цене (от/до) объявления не фильтруются корректно. В результатах отображаются объявления вне заданного диапазона.

**Шаги воспроизведения:**
1. Открыть главную страницу `/`
2. В поле «От» ввести: 5000
3. В поле «До» ввести: 15000
4. Дождаться применения фильтра 
5. Проверить цены отображаемых объявлений

**Фактический результат:**

Отображаются объявления с ценами вне диапазона 5000-15000 

**Ожидаемый результат:**

Должны отображаться только объявления с ценой от 5000 до 15000

**Серьёзность:**

Major 

**Приоритет:**

P0

**Окружение:**

`https://cerulean-praline-8e5aa6.netlify.app`

# BUG-UI-002: Поле цены принимает отрицательные значения
**Краткое описание:**

Поле ввода минимальной/максимальной цены принимает отрицательные значения.

**Шаги воспроизведения:**
1. Открыть главную страницу `/`
2. В поле «От» ввести: `-100`
3. Дождаться валидации 
4. Проверить значение в поле

**Фактический результат:**

Поле сохраняет отрицательное значение `-100`

**Ожидаемый результат:**

Поле должно:
* Либо отклонить ввод отрицательного числа
* Либо автоматически заменить на `0`

**Серьёзность:**

Medium

**Приоритет:** 

P1

**Окружение:**

`https://cerulean-praline-8e5aa6.netlify.app`

# BUG-UI-003: Поле цены не ограничивает длину ввода
**Краткое описание:**

Поле ввода цены принимает неограниченное количество цифр (30+), что может привести к переполнению или некорректной обработке.

**Шаги воспроизведения:**
1. Открыть главную страницу `/`
2. В поле «От» ввести: `999999999999999999999999999999` (30 девяток)
3. Проверить, что осталось в поле
   
**Фактический результат:**

Поле принимает и сохраняет все 30+ цифр без ограничения.

**Ожидаемый результат:**

Поле должно ограничивать ввод:
* Максимум 10-12 цифр (для цен до 999 999 999 999)

**Серьёзность:**

Medium

**Приоритет:** 

P2

**Окружение:**

`https://cerulean-praline-8e5aa6.netlify.app`
